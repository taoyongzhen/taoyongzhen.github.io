HashMap源码分析（1.7版本）

发表于 2017-08-06 | 分类于 Java ， Map

概述

HashMap是一个用”KEY”-“VALUE”来实现数据存储的类。可以依据”key”存储数据。也可以通过”key”去得到数据。所以你可以把HashMap当作一个字典。
由于HashMap实现方式在jdk1.7和jdk1.8版本间做了较大修改，本文以jdk1.7版本展开分析，下一篇日志以jdk1.8版本分析，并对他们做比较、总结。

基本要点

设计理念

HashMap是一种基于哈希表（hash table）实现的map，哈希表（也叫关联数组）一种通用的数据结构。其定义：通过把关键码值映射到表中一个位置来访问记录，以加快查找的速度。这个映射函数叫做散列函数，存放记录的数组叫做散列表。
其要素有二：

  • 定义key到位置的映射函数，尽可能地使其均匀分布散列表；
  • 发生冲突后解决方法。

Hash函数包括两种：

  • 让存储数组的长度Length为素数，然后用hashCode(key) mod length的方法得到索引；
  • 让存储数组的长度Length为2的指数幂，然后用hashCode(key) & (length-1)的方法得到索引
    注：HashTable是用第一种方法；文本介绍的主角HashMap使用第二种方法。
    冲突解决最为常用的链地址法。即用一个列表保存所有冲突数据，HashMap和HashTable均使用该方法。

Length为2的指数幂原因：

  • 便于进行位运算，而位运算的效率高于普通运算；
  • 由于Length为2的指数幂，length-1最后一位为1，这样便保证了h&(length-1)的最后一位可能为0，也可能为1（这取决于h的值），即与操作后的结果可能为偶数，也可能为奇数，这样便可以保证散列的均匀性；而如果length为奇数的话，则length-1为偶数，它的最后一位是0，这样h&(length-1)的最后一位肯定为0，即只能为偶数，这样任何hash值都只会被散列到数组的偶数下标位置上，这便浪费了近一半的空间。

总结

  • 非线程安全，多线程环境下，可能出现死循环，详细解释见参考4.
  • fail-fast错误。多线程环境下遍历的同时，进行结构性修改可能会引发fail-fast错误。
  • 允许key与value都为null值，HashTable与之相反，为线程安全，key与value都不允许null值。
  • 不保证存储顺序。随着时间的推移，同一元素的位置也可能改变（resize的情况）
  • put、get操作的时间复杂度为O(1+n)。
  • 依据集合视角进行遍历。时间复杂度与其容量和元素个数成正比，所以如果遍历的性能要求很高，不要把capactiy设置的过高或把平衡因子（load factor，capactiy设置过高，元素均匀分配在各个桶内，导致列表切换次数增加；load factor过低，相同元素个数下会导致capactiy变大，进而导致遍历时间增加。

LinkedList源码分析

发表于 2017-08-03 | 分类于 Java ， List

概述

LinkedList是线性表中的链式存储结构的顺序表。

基本元素

类申明

  • 继承AbstractSequentialList，获取list基本操作
  • 实现List接口（重复）表明自身属于List
  • 实现Deque接口，其可以作为双向队列使用（介绍见附录）
  • 实现Cloneable接口，重写clone() ，能克隆(浅拷贝)
  • 实现Serializable接口，支持序列化

构造函数

上述空构造函数、集合类参数构造函数是集合类构造函数的标配，各种集合类都具有上述构造函数。

常量解释

  • size：LinkedList包含元素个数
  • capacity：一般是指为容器类分配的存储单元的个数，且capacity> =size。具体到LinkedList，由于存储结构为链表，实现按需分配，动态释放，所以capacity=size。
  • first：头结点，指向第一个元素节点，是个引用。
  • last：尾节点，指向最后一个元素节点，是个引用。
  • modCount：结构性修改次数，fast-fail错误的判断依据
  • Node：节点，代表LinkedList元素的结构

基本操作

增加操作

  • add(int index, E element)

查询操作

查询操作涉及到的函数：List接口具有的public E get(int index)，Queue接口以及子接口的：getFirst()，getLast()，peek()，peekLast()，peekLast()，peekLast()。

  • get(int index)

总结

  • 由于继承于Queue及其子接口的双向链表，可以被当作堆栈、队列或双端队列进行操作
  • 有序，双向链表，默认使用尾部插入法。维护数据的有序插入
  • 非线程安全，注意fast-fail错误
  • 适用于删除、和增加操作，时间复杂度为O(1)频繁访问请选用ArrayList
  • 允许插入NUUU元素，但是不建议，使用NULL表示插入等操作失败

附录

顺序存储和链式存储

顺序存储：存储位置连续，可以很方便获取各个元素的地址

优点：查询很快
缺点：插入和删除效率慢

链式存储：用一组任意的存储单元存储线性表的数据元素，这组存储单元可以是连续的，也可以是不连续的。

优点：插入和删除效率高
缺点：查询效率低

Queue接口

概述

Java集合框架下的Queue接口继承自Collection接口 ，Deque, LinkedList,PriorityQueue,BlockingQueue等类都实现了它。
Queue主要用来处理元素的集合。

基本方法

Queue还添加了额外的 添加、删除、查询操作，主要包含两种形式，一种操作失败时直接抛出异常，而另一种则返回一个特殊的值。

Remove remove() poll(e)
EXamine element(e) peek(e)

add(e)和offer(e)出于区分remove等函数执行成功与否，java建议不要添加NULL元素。在其实现中，禁止添加NULL时，加入NULL就会报空指针异常。Queue的实现类中，比如ArrayBlockingQueue，PriorityBlockingQueue 等等响应了这一建议；比如LinkedList等没有响应。

Deque接口

概述

Deque是双向队列,Deque继承自Queue,直接实现了它的有LinkedList, ArayDeque,ConcurrentLinkedDeque等。

基本方法

Deque接口定义了一些从头部和尾部访问元素的方法。比如分别在头部、尾部进行插入、删除、获取元素。它的方法返回类型和Queue相同。

Deque继承了Queue接口的方法。当Deque作为队列使用时（FIFO），添加元素是添加到队尾，删除时删除的是头部元素。

Deque可以当栈用（后进先出）。这时入栈、出栈元素都是在双向队列的首节点进行。

Deque 的实现类根据线程安全主要分为两种：

ArrayList源码分析

发表于 2017-08-01 | 分类于 Java ， List

概述

ArrayList基于数组实现，使用一个动态数组保存数据，其容量自动增加。

名词解释

  • DEFAULT_CAPACITY：默认初始容量10，向其添加元素后，创建默认大小为10的数组。
  • elementData：ArrayList使用该数组保存元素，使用transient关键字修饰，在序列化时，将该字段忽略。
  • EMPTY_ELEMENTDATA：空数组，创建大小为0的ArrayList时，使用其初始化elementData。
  • DEFAULTCAPACITY_EMPTY_ELEMENTDATA：空数组，使用缺省的构造函数时，使用其初始化elementData。与EMPTY_ELEMENTDATA区别是：ArrayList添加第一个元素时，将其容量扩充到DEFAULT_CAPACITY。这里需要注意：使用延迟初始化元素的存储数组，已达到内存优化。
  • size：元素个数。
  • modCount：结构性修改次数，结构性修改是指改变元素个数的修改，例如add等操作。

基本操作

增加操作

  • 列表结尾添加
  • 指定位置添加

删除操作

  • 指定位置删除
  • 指定元素删除
    注意对NULL元素的处理（侧面说明支持NULL元素的加入）；删除所有指定元素。
  • 集合删除
    删除包含在指定集合中所有元素。
  • 集合保留
    保留指定集合中的所有元素。

上述集合类操作设计到函数：batchRemove(Collection<?> c, boolean complement)。它在O(n)下完成数组的过滤，算法值得学习。

修改

完成指定位置元素的替换（非结构性修改）

查找

获取指定位置元素。

扩容操作

需要明确一下几点：

  • 初始默认容量为10；
  • 按需扩容：MAX(old_capacity*1.5,size + add_elements);
  • 可以预估元素个数，使用ArrayList(int initialCapacity)，避免扩容。

    fail-fast

    描述

    fail-fast也就是“快速失败”，它是Java集合的一种错误检测机制。当多个线程对集合进行结构上的改变的操作时，有可能会产生fail-fast错误。

    原因分析

    fail-fast产生的原因就在于多线程环境下，部分线程在对 collection 进行迭代时，部分线程对该 collection 进行结构性修改，这时迭代器就会抛出 ConcurrentModificationException 异常信息，从而产生 fail-fast。

解决方法

  • 单线程环境下，可以使用迭代器提供的remove、add方法；
  • 使用CopyOnWriteArrayList来替换ArrayList；
  • 使用List list = Collections.synchronizedList(new ArrayList(…))方法。

总结

  • ArrayList是线性表中的顺序存储结构的顺序表，因为内部通过个数组保存元素，数组是一个拥有连续存储地址的存储块。
  • ArrayList因为内部维护的是一个数组，查询和修改的效率很高，但是插入添加和删除操作涉及元素迁移，效率比较低下。
  • ArrayList在允许加入NULL元素。
  • ArrayList从允许添加重复的元素对象。
  • ArrayList非线程安全，线程安全使用使用CopyOnWriteArrayList、Vector。
  • ArrayList支持序列化，基本思想是：写入时，序列化元素个数，依次序列化各个元素；反之亦然。

Java 集合类概述

发表于 2017-07-29 | 分类于 Collection

框架概述

首先需要理解集合的概念，集合是指由同类元素的整体。而java集合是指java提供的一套工具包，以方便用户的使用，包含了常用的数据结构：队列、链表、栈等。Java集合框架是指实现该工具包的一套规范，用来表示、操作集合，用以方便使用者理解、使用其中的数据结构。
学习、使用每一类集合时，需要从以下几个维度进行把握：

  • 基本操作：增删改查，注意支持重复元素、NULL元素处理等。
  • 线程安全：是不是支持多线程，以及实现多线程安全的方式。
  • 效率: 空间复杂度和时间复杂度，包括基本操作的时间复杂度等。

    设计理念

    主要的设计理念：提供一套小而美的API。能够友好支持新增数据结构，降低程序员的学习成本。
    为了保证核心接口足够小、具有普适性，并不会区分该集合是否可变（mutability）,是否可更改（modifiability）,是否可改变大小（resizability）这些细微。例如，部分接口不支持某类操作，在是在实现时抛出UnsupportedOperationException即可表示集合不支持该操作。

    框架类图

    Java集合主要可以划分为4个部分：List列表、Set集合、Map映射、工具类(Iterator迭代器、Enumeration枚举类、Arrays和Collections)。（下图来自网络).

    下面依次简单介绍Collection和Map接口，其中Collection介绍List、Set和Queue接口。

    Collection接口

    Collection接口作为集合类顶级接口，除了定义了基本的增删改查操作，还添加了clear()、size()等基本方法。这也符合其设计理念—定义一套小而全、具有普适作用的API。它是Queue、List和Set的父接口，包含他们公共的方法。

    List接口

    List接口通常表示一个列表（数组、队列、链表、栈等），其中的元素有序并且可以重复，常用实现类为ArrayList和LinkedList，另外还有不常用的Vector。
  • AbstractCollection:针对顶级接口类Collection中成员方法提供部分实现，增加Collection子类代码复用。
  • List：有序集合的顶层接口，定义一些基本方法。
  • AbstractList：针对顶级有序集合接口类List中成员方法提供部分实现，增加List子类代码复用。
  • AbstractSequentialList：在AbstractList基础上，提供顺序访问（数据存储方式为列表，有别于数据存储方式为数据，访问方式随机）。

    Set接口

    Set接口通常表示一个集合，其中的元素不允许重复（通过hashcode和equals函数保证），常用实现类有HashSet和TreeSet。
  • AbstractSet:针对接口类Set中成员方法提供实现，增加Set子类代码复用。
  • SortedSet:在接口类Set基础上定义有序访问。
  • NavigableSet：在接口类SortedSet基础上进行扩展。

    Queue接口

    Map接口

    Map是一个映射接口，其中的每个元素都是一个key-value键值对，能够实现根据key快速查找value。
  • AbstractMap:针对接口类Map中成员方法提供实现，增加Map子类代码复用。
  • NavigableMap：在接口类SortedMap基础上进行扩展。
  • SortedMap:在接口类Map基础上定义有序访问。
  • ConcurrentMap：在接口类Map基础上定义线程安全和原子访问
  • ConcurrentNavigableMap：在接口类ConcurrentMap基础上，添加NavigableMap方法
  • Dictionary:是一个抽象类，用来存储键/值对，作用和Map类相似
    Map并不是一个真正意义上的集合（are not true collections），但是这个接口提供了三种“集合视角”（collection views ），使得可以像操作集合一样操作它们，具体如下：
  • 把map的内容看作key的集合
  • 把map的内容看作value的集合
  • 把map的内容看作key-value映射的集合

    总结

  • 每种集合实现形式，定义了数据独特的“增删改查”方式。
  • 使用者要依据场景，在空间复杂度和时间复杂度做出平衡的基础上，选择最为合适的数据集合。
